# 问卷系统（AI 编程评估）

## 概述

构建一个 **基于 Schema 驱动的问卷系统**。这是一个以前端为主的评估项目。

系统包含两个前端应用：

1. **管理端界面** – 创建和配置问卷（题目、选项、逻辑）
2. **用户端界面（运行时）** – 解析问卷 Schema，为最终用户渲染可交互的问卷，包括动态行为

> **不在要求范围内：** 无需实现答案提交或持久化。重点仅在于 **定义与解析问卷 Schema**，以及 **在运行时正确渲染并控制问题流程**。

---

## 仓库结构与开始方式

1. 克隆评估仓库：
   ```
   git clone git@github.com:neutronsg/engineering-assessment-fe.git
   cd engineering-assessment-fe
   ```

2. 仓库包含以下目录：
   ```
   backend/             # 你的后端实现（如需）
   backend_example/     # 示例后端，供参考
   frontend/
      admin/            # 管理端应用
      web/              # 用户端运行时应用
   README.md
   README.zh-CN.md
   ```

3. 在现有结构中完成实现：
   - 将 **管理端界面** 代码放在 `frontend/admin` 目录下
   - 将 **用户端界面（运行时）** 代码放在 `frontend/web` 目录下
   - 如果需要，可以使用 `backend/` 目录搭建数据存储层，也可以选择其他方式（见后端部分说明）

4. 将完成的代码推送到一个 **公开仓库**，并提供仓库 URL 供评审。

## 核心要求

至少需要实现以下能力：

- **问卷编写** – 支持创建和配置以下题型：
  - 单选题（radio）
  - 多选题（checkbox）
  - 自由文本
- **动态问卷流** – 问题的展示顺序与可见性根据用户答案动态调整。例如：根据答案分支到不同的后续问题、跳过无关部分，或结合多个答案共同决定接下来出现哪些问题（如评分、多条件分支）。你需要自己设计驱动这一行为的规则引擎和数据模型。
- **运行时解析与渲染** – 用户端应用读取问卷定义（Schema），并生成一个遵循所定义逻辑的、可工作的交互式表单。

深度优先于广度。请专注于使核心路径稳健且架构良好，而非堆砌大量功能。

## 设计自主权

以下内容有意未作规定——你需要自行设计：

- 问卷定义的数据模型 / Schema
- 用于动态流的规则引擎（规则如何定义、存储和求值）
- UI 组件树与布局
- 状态管理方案

我们关注的是 **你如何处理一个模糊的问题**、如何划定边界，以及如何产出一个清晰的架构，以便 AI 能够在此基础上帮助你高效实现。

---

## AI 使用要求

这是一个 **AI 辅助的评估项目**。你需要全程使用 AI 工具（如 ChatGPT、Claude Code、Copilot、Cursor、Codex 等）协同工作。

本任务的复杂度和开放性决定了，你几乎必须借助 AI 才能在合理的时间内完成。你应当专注于：

- 定义正确的问题和约束条件
- 设计架构与组件拆分
- 用精确的指令和上下文引导 AI
- 评审、精炼和整合 AI 生成的代码

我们将评估你 **与 AI 协作的能力**，而不仅仅是你的裸编码速度。

---

## 后端与数据持久化

对数据存储方式没有任何限制。你可以使用任何能让你存取问卷定义的方式（REST API、本地 JSON、模拟服务等）。仓库中提供了 `backend/` 目录用于构建简单的存储服务（如果需要），`backend_example/` 作为参考。后端**仅作为数据存储托管服务**，不得包含任何业务逻辑（如校验、评分、分支规则等）。所有逻辑都必须在前端完成。后端的实现细节不纳入评估，只有数据契约作为你前端架构的一部分会被关注。

---

## 评估重点

我们将评估：

- 你对问题的 **拆解** 能力和解决方案的结构化程度
- 你设计的 **架构与数据模型**
- 你如何借助 AI 探索权衡方案并生成健壮的代码
- 你与 **AI 协作的质量**（清晰的指引、迭代式优化、错误处理）
- **动态问卷流** 的正确性与响应性
